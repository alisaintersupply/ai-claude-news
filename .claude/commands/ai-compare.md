Compare two or more AI models, companies, or approaches side by side.

Usage: /ai-compare [A] vs [B]

Examples:
- /ai-compare GPT-4o vs Claude 3.7
- /ai-compare RAG vs fine-tuning
- /ai-compare OpenAI vs Anthropic (company comparison)
- /ai-compare Llama vs Mistral (open source models)

Comparison dimensions (select relevant ones):
1. **Capabilities** — What each can do, benchmark scores
2. **Cost** — Pricing per million tokens, API costs
3. **Speed** — Latency, throughput benchmarks
4. **Context window** — Token limits
5. **Availability** — API access, regional restrictions
6. **Safety** — Content policies, refusal rates
7. **Use cases** — Best fit scenarios for each

Output format:
- Summary comparison table
- Detailed analysis per dimension
- Thai verdict: "เหมาะสำหรับ..." (best for...)
- When to choose A vs B

Always use current data — state your knowledge cutoff clearly.
